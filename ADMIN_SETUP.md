# Налаштування доступу до адмін панелі

## Важливо

Адмін панель доступна **тільки одному користувачу** - замовнику (власнику проекту).

## Як налаштувати

1. Відкрийте файл `.env` в кореневій директорії проекту.

2. Додайте або оновіть змінну `ADMIN_EMAIL` з email замовника:

```env
ADMIN_EMAIL=your-owner-email@example.com
```

3. Перезапустіть backend сервер.

## Як це працює

- Тільки користувач з email, вказаним в `ADMIN_EMAIL`, може отримати доступ до адмін панелі.
- Користувач має бути зареєстрований через email/пароль (не через Google OAuth).
- Користувач має мати роль `super_admin` в базі даних.

## Створення користувача-замовника

Якщо вам потрібно створити користувача-замовника:

```bash
node scripts/create-admin.js owner@example.com your-secure-password
```

Ця команда створить користувача з:
- Email: `owner@example.com`
- Роль: `super_admin`
- Link limit: 9999 (необмежений)

**Після створення користувача, встановіть `ADMIN_EMAIL=owner@example.com` в `.env` файлі.**

## Безпека

- Якщо `ADMIN_EMAIL` не встановлений в `.env`, доступ до адмін панелі будуть мати всі користувачі з роллю `super_admin`.
- Рекомендується **завжди** встановлювати `ADMIN_EMAIL` в продакшені.
- Email порівнюється без урахування регістру (case-insensitive).
