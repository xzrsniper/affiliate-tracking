# Допомога з підключенням до VPS через SSH

## Проблема: Permission denied при підключенні

### Варіант 1: Перевірка даних для входу

**Можливі причини:**
1. Неправильний пароль
2. Неправильне ім'я користувача
3. Потрібно використовувати SSH ключі

**Що перевірити:**
- Ім'я користувача може бути не `ergoa`, а інше (наприклад, `root`, `admin`, `ubuntu`, `debian`)
- Пароль може містити спеціальні символи, які потрібно вводити уважно
- Деякі хостинги вимагають використання SSH ключів замість пароля

### Варіант 2: Спробуйте інші імена користувачів

```bash
# Спробуйте root
ssh root@vps76168.hyperhost.name

# Або admin
ssh admin@vps76168.hyperhost.name

# Або ubuntu (для Ubuntu серверів)
ssh ubuntu@vps76168.hyperhost.name
```

### Варіант 3: Перевірте пароль в панелі управління хостингом

1. Увійдіть в панель управління HyperHost
2. Перевірте дані для SSH доступу
3. Можливо, потрібно скинути пароль або створити нового користувача

### Варіант 4: Використання SSH ключів (рекомендовано)

**Якщо у вас є приватний ключ:**

```bash
# Windows PowerShell
ssh -i C:\path\to\your\private_key ergoa@vps76168.hyperhost.name

# Або якщо ключ в стандартній папці
ssh -i ~/.ssh/id_rsa ergoa@vps76168.hyperhost.name
```

**Якщо ключа немає, створіть його:**

```powershell
# На Windows (PowerShell)
ssh-keygen -t rsa -b 4096

# Ключ буде створено в C:\Users\ergoa\.ssh\id_rsa
# Публічний ключ: C:\Users\ergoa\.ssh\id_rsa.pub
```

**Додайте публічний ключ на сервер:**
1. Скопіюйте вміст файлу `id_rsa.pub`
2. У панелі HyperHost додайте цей ключ до SSH ключів
3. Або якщо є доступ до сервера іншим способом, додайте в `~/.ssh/authorized_keys`

### Варіант 5: Підключення через панель управління

Деякі хостинги надають веб-термінал:
1. Увійдіть в панель HyperHost
2. Знайдіть "Terminal" або "SSH Console"
3. Використайте веб-термінал для підключення

### Варіант 6: Скидання пароля через панель

1. У панелі HyperHost знайдіть "SSH Access" або "Server Management"
2. Скиньте пароль для користувача
3. Або створіть нового користувача з SSH доступом

## Альтернативні способи завантаження файлів

### Якщо SSH не працює, використайте SFTP:

**Через FileZilla:**
1. Встановіть FileZilla
2. Host: `vps76168.hyperhost.name`
3. Username: `ergoa` (або інше)
4. Password: ваш пароль
5. Port: `22`
6. Protocol: `SFTP`

**Через WinSCP:**
1. Встановіть WinSCP
2. File protocol: `SFTP`
3. Host name: `vps76168.hyperhost.name`
4. User name: `ergoa`
5. Password: ваш пароль

## Після успішного підключення

Коли з'єднання встановлено, виконайте:

```bash
# Перевірте версію Node.js (має бути 18+)
node --version

# Якщо Node.js не встановлено, встановіть:
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# Перевірте MySQL
mysql --version

# Якщо MySQL не встановлено:
sudo apt update
sudo apt install -y mysql-server
```

## Контакти для підтримки

Якщо проблема залишається:
1. Зверніться в підтримку HyperHost
2. Запитайте правильні дані для SSH доступу
3. Перевірте, чи увімкнений SSH доступ для вашого акаунту
