# Як викласти Shopin (shopin.pp.ua) без хостингу — тільки домен

Якщо є **тільки домен** shopin.pp.ua і немає хостингу, сайт можна безкоштовно розмістити на одному з сервісів нижче. Усі вони дають безкоштовний SSL (https) і підтримку свого домену.

---

## Варіант 1: Netlify (найпростіший, без реєстрації в Git)

**Час: 5–10 хвилин.**

1. **Зареєструйтесь:** [https://www.netlify.com](https://www.netlify.com) (можна через Google/GitHub/email).

2. **Створіть сайт:**
   - Увійдіть у Netlify → **Add new site** → **Deploy manually**.
   - У вікно **"Drag and drop your site output folder here"** перетягніть **папку** `shopin-pp-ua` з вашого комп’ютера (та, де лежать `index.html`, `checkout.html`, `thank-you.html`).
   - Зачекайте, поки з’явиться посилання типу `https://random-name-123.netlify.app` — це вже працюючий сайт.

3. **Підключіть свій домен shopin.pp.ua:**
   - Відкрийте сайт у Netlify → **Domain settings** → **Add custom domain**.
   - Введіть: `shopin.pp.ua` (і за бажанням `www.shopin.pp.ua`).
   - Netlify покаже, які записи додати у вашому реєстраторі домену.

4. **Налаштування домену у реєстратора (де купували shopin.pp.ua):**
   - Зайдіть у панель управління доменом (де купували .pp.ua).
   - Знайдіть розділ **DNS** (або «Управління DNS», «Зона домену»).
   - Додайте запис типу **A** або **CNAME**, як написано в Netlify:
     - Зазвичай для кореня домену: **A** запис на `75.2.60.5` (або IP, який покаже Netlify).
     - Для **www**: **CNAME** запис `www` → `ваш-сайт.netlify.app`.
   - Збережіть зміни. Оновлення DNS займає від кількох хвилин до 24–48 годин.

5. **SSL у Netlify:** Після того як домен підтягнеться, Netlify автоматично видасть сертифікат — сайт буде відкриватися по **https://shopin.pp.ua**.

**Підсумок:** хостингу не потрібно — тільки домен і обліковий запис Netlify. Сайт = статичні файли з папки `shopin-pp-ua`.

---

## Варіант 2: Cloudflare Pages (безкоштовно, швидко, ваш домен у Cloudflare)

**Час: 10–15 хвилин.**

1. **Реєстрація:** [https://pages.cloudflare.com](https://pages.cloudflare.com) або [https://dash.cloudflare.com](https://dash.cloudflare.com).

2. **Створення проєкту:**
   - **Create a project** → **Direct Upload**.
   - Завантажте **архів (ZIP)** з трьома файлами: `index.html`, `checkout.html`, `thank-you.html` (можна запакувати папку `shopin-pp-ua`).
   - Далі натисніть **Deploy site**. З’явиться адреса типу `ваш-проєкт.pages.dev`.

3. **Підключення домену shopin.pp.ua:**
   - Якщо домен ще не в Cloudflare: у **Cloudflare** додайте сайт (Add a site) і вкажіть `shopin.pp.ua`; змініть NS-записи у реєстратора на ті, що дасть Cloudflare.
   - У **Pages** → ваш проєкт → **Custom domains** → **Set up a custom domain** → введіть `shopin.pp.ua` (і при потребі `www.shopin.pp.ua`).
   - Cloudflare сам підставить потрібні DNS-записи, якщо домен обслуговується в Cloudflare.

4. **SSL:** У Cloudflare SSL зазвичай увімкнений за замовчуванням (Flexible або Full) — сайт буде по **https://shopin.pp.ua**.

**Підсумок:** хостингу не потрібно; якщо перенесете домен в Cloudflare, і хостинг, і DNS будуть у одному місці.

---

## Варіант 3: GitHub Pages (якщо є або не важко створити GitHub)

**Час: 10–20 хвилин.**

1. **Акаунт GitHub:** [https://github.com](https://github.com) — зареєструйтесь, якщо його немає.

2. **Новий репозиторій:**
   - **New repository**, назва наприклад `shopin-pp-ua`.
   - Публічний, без README — **Create repository**.

3. **Завантаження файлів:**
   - У репозиторії натисніть **Add file** → **Upload files**.
   - Перетягніть у вікно браузера файли: `index.html`, `checkout.html`, `thank-you.html` з папки `shopin-pp-ua`.
   - **Commit changes**.

4. **Увімкнення GitHub Pages:**
   - Репозиторій → **Settings** → зліва **Pages**.
   - **Source:** Deploy from a branch.
   - **Branch:** `main` (або `master`), папка **/ (root)** → **Save**.
   - Через хвилину сайт буде доступний за адресою `https://ваш-логін.github.io/shopin-pp-ua/`.

5. **Свій домен shopin.pp.ua:**
   - У тій же вкладці **Pages** у полі **Custom domain** введіть `shopin.pp.ua` → **Save**.
   - У реєстраторі домену в DNS додайте:
     - **A** записи на IP GitHub: `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
     - або **CNAME** запис: `shopin.pp.ua` → `ваш-логін.github.io` (якщо реєстратор дозволяє CNAME для кореня).
   - У **Pages** можна увімкнути **Enforce HTTPS**.

**Підсумок:** хостингу не потрібно — тільки GitHub-акаунт і налаштування DNS для shopin.pp.ua.

---

## Що змінити в DNS у реєстратора (загальна схема)

Де б ви не розмістили сайт (Netlify, Cloudflare Pages, GitHub), у панелі **домену shopin.pp.ua** потрібно:

- Або додати **A**-записи на IP, який дасть сервіс.
- Або додати **CNAME** (наприклад, `www` → адреса типу `сайт.netlify.app` або `ваш-логін.github.io`).

Точні значення завжди показує сам сервіс (Netlify / Cloudflare / GitHub) у розділі «Custom domain» або «DNS».

---

## Після викладення

- Відкрийте **https://shopin.pp.ua** (або тимчасову адресу типу .netlify.app / .pages.dev) і переконайтесь, що відкривається головна сторінка магазину.
- Тест трекера: відкрийте `https://shopin.pp.ua/?ref=ВАШ_КОД`, зробіть тестове замовлення і перевірте конверсію в Lehko Track.
- У Lehko Track у **Налаштування** → **Мої сайти** додайте домен `shopin.pp.ua` — через кілька хвилин статус має стати «Підключено».

Якщо напишете, яким реєстратором користуєтесь для .pp.ua, можна розписати кроки саме для його панелі (де натиснути, які поля заповнити).
